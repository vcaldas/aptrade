Sizer
=====

The ``Sizer`` component defines how much capital or how many units are
allocated to each order generated by a strategy. It is responsible for
turning a trading signal into a concrete position size while respecting the
portfolio and risk constraints configured in the system.

Overview
--------

A sizer typically receives the current execution context and returns the size
to use for a new order. This allows strategies to stay focused on entry and
exit logic while sizing logic remains isolated and reusable.

Depending on the implementation, a sizer may consider:

* available cash
* current position exposure
* instrument price
* fixed quantity rules
* percentage-based allocation
* account or instrument risk limits

Common sizing approaches
------------------------

Fixed size
~~~~~~~~~~

Always submits the same quantity for every order.

Use this approach when you want predictable and simple behavior, especially
for testing or baseline strategy evaluation.

Percentage of capital
~~~~~~~~~~~~~~~~~~~~~

Allocates a percentage of available capital to each trade.

This approach scales position size with account growth or drawdown and is a
common choice for portfolio-aware strategies.

Risk-based sizing
~~~~~~~~~~~~~~~~~

Determines order size from a target risk amount, often using the distance
between entry price and stop-loss.

This is useful when strategies aim to normalize risk across trades instead of
using the same raw quantity for every instrument.

Responsibilities
----------------

The sizer is expected to:

* compute an order quantity for buy or sell operations
* enforce sizing constraints before order submission
* provide deterministic behavior for the same portfolio state and inputs

In general, the sizer should not contain entry or exit signal logic. Its role
is limited to position sizing.

When to use a custom sizer
--------------------------

Create a custom sizer when the default sizing behavior does not match your
strategy requirements, such as:

* dynamic allocation across multiple assets
* volatility-adjusted position sizing
* maximum exposure per symbol or sector
* leverage-aware sizing rules

Best practices
--------------

* Keep sizing rules simple and transparent.
* Validate edge cases such as zero price, insufficient cash, or missing stop
	levels.
* Ensure the returned size is consistent with broker and instrument
	constraints.
* Test the sizer independently from the strategy logic.

Summary
-------

The ``Sizer`` component centralizes position sizing decisions and helps make
strategies safer, more reusable, and easier to maintain.
